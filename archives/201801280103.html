    博客回顾－－－virtualenv

    在博客环境搭建之初，碰到了很多的问题，在一些博主的文章指导下，对pelican搭建博客有了一定的了解。

    如下做一些简单的纪录，以备后续博客维护以及修改用。

    刚开始想在环境下直接安装一堆的东西，但想到自己环境里面同时装了python2.7 以及python3.4，同时pip也只装了3的，所以在操作过程中太麻烦，平常在使用的过程中也分为python3 和pip3 使用，而2.7 才是用python，但是却没有装pip，所以这次，我们先把pip装上:

    sudo easy_install pip

    想到要在环境上要装那么多packet，会把自己的环境搞的非常的复杂，偶然在某个博主的搭建过程中看到了一个virtualenv 工具，可以把环境独立出来，可以尝试一下：

    //首先安装virtualenv
sudo pip install virtualenv
//创建一个虚拟python环境，virtualenv实际上就是设置了一个环境，其中环境变量全部于系统中的独立开来，所以开辟的空间时非常干净的，我们可以利用这个，在独立的空间里面安装我们自己想要的包。
virtualenv myBlogEnv
//开始使用这个虚拟环境
cd myBlogEnv
source ./bin/activate
//然后就会在命令行的前面多了一个括号内容为myBlogEnv，就可以开始使用这个环境了。要退出则直接去激活
deactivate

    至此，virtualenv就完成了，后续在在这个虚拟环境中直接pip安装软件就可以了。
